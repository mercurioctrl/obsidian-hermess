# PED-1388 · API PED fix  pago por trasnferencia bancaria 

[PED-1388 en Jira](https://bluinc.atlassian.net/browse/PED-1388)

## Descripción

```php
# Payment For Bank
Route::post('paymentForBank', PaymentForBank::class);
```



### Resumen

Al autorizar pedidos con importe mayor a 999 USD, el movimiento en `MC_CCORRIENTES_MOVIMIENTOS` se registraba con un valor incorrecto.    
Por ejemplo, `2005.45000` se guardaba como `2`.

### Contexto

La causa era el uso de `number_format()` casteado a `float`.    
La coma del separador de miles hacía que PHP truncara el valor.

### Criterios de aceptación

- Corregir el registro del movimiento en `MC_CCORRIENTES_MOVIMIENTOS` para importes mayores a 999 USD.
- Reemplazar `number_format()` por `round()` en:  - `ExecuteTradeBank`
  - `PaymentForBankService`
  - `BankValidation`
