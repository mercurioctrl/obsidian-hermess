# LIO-632 · API - Review - Categoria "Cables" y "Cables y Adaptadores" es ambigua y parece estar mal catalogada

[LIO-632 en Jira](https://bluinc.atlassian.net/browse/LIO-632)

## Descripción

Basicamente navegando encontramos 2 categorias

https://gamma.libreopcion.com/productos?o=rel&categorias=cables_y_adaptadores

```
https://gamma.libreopcion.com/productos?o=rel&categorias=cables_y_adaptadores
```

y

https://gamma.libreopcion.com/productos?o=rel&categorias=cables

```
https://gamma.libreopcion.com/productos?o=rel&categorias=cables
```



Si bien ambas tienen dentro productos que no coinciden 100% deberia usarse de la siguiente manera

Componentes de pc > Cables y Adaptadores: Aca deberia ir todo lo que son cables y adaptadores de uso para PC

Hogar → Electricidad > Cables : Esta categoria padre se inagura nueva para darl lugar a un monton de categorias que van dentro de electricidad. Aca van los cables electricos tipo https://gamma.libreopcion.com/productos?o=rel&categorias=cables_y_adaptadores

Actualmente la categoría **Cables y adaptadores** esta mezclando muchas categorias de cables de electricidad y cables de pc, la idea es poder separar esta mezcla de cables donde quede algo asi:

```
Componentes de PC
└── Cables y Adaptadores
    ├── HDMI
    ├── USB
    ├── UTP
    ├── DisplayPort
    ├── SATA
    ├── Audio
    └── Adaptadores

Electricidad
└── Cables
    ├── Unipolar
    ├── Taller
    ├── Subterráneo
    ├── TPR
    ├── Alta tensión
    └── Instalación eléctrica
```

### @Franco Callipo — 2026-05-19 11:33:26

Con esta query se ven los id de las categorias padre

```
SELECT id, nombre
FROM LO.dbo.categorias
WHERE nombre LIKE '%Cables%';

id          nombre              
----------  --------------------
23          Cables              
136         Accesorios y cables 
138         Adaptadores y cables
279         Cables y adaptadores
```

Con esta query se ven los productos que no deberian estar en la categoria de pc porque son de la categoría electricidad

```
SELECT 
    p.id,
    p.titulo,
    c.nombre AS categoria_actual
FROM CS.dbo.productos p
INNER JOIN LO.dbo.categorias c
    ON c.id = p.id_categoria_lo
WHERE p.id_categoria_lo = 279
AND (
       p.titulo LIKE '%UNIPOLAR%'
    OR p.titulo LIKE '%SUBTERRANEO%'
    OR p.titulo LIKE '%TALLER%'
    OR p.titulo LIKE '%TPR%'
    OR p.titulo LIKE '%TRP%'
    OR p.titulo LIKE '%ALTA TENSION%'
    OR p.titulo LIKE '%CERCO ELECTRICO%'
    OR p.titulo LIKE '%TREFILCON%'
    OR p.titulo LIKE '%ARGENPLAS%'
)
ORDER BY p.titulo;
```

Con esta query se ven los productos que no deberian estar en la categoria electricidad porque son de la categoría de pc

```
SELECT 
    p.id,
    p.titulo,
    c.nombre AS categoria_actual
FROM CS.dbo.productos p
INNER JOIN LO.dbo.categorias c
    ON c.id = p.id_categoria_lo
WHERE p.id_categoria_lo = 23
AND (
       p.titulo LIKE '%HDMI%'
    OR p.titulo LIKE '%USB%'
    OR p.titulo LIKE '%DISPLAYPORT%'
    OR p.titulo LIKE '%VGA%'
    OR p.titulo LIKE '%SATA%'
    OR p.titulo LIKE '%UTP%'
    OR p.titulo LIKE '%PATCH%'
    OR p.titulo LIKE '%RED%'
)
ORDER BY p.titulo;
```
