# LIO-639 · API - Feat - Al realizar una compra por "transferencia bancaria" se debe enviar un recordatorio que permita hacer la transferencia.

[LIO-639 en Jira](https://bluinc.atlassian.net/browse/LIO-639)

## Descripción

### Resumen

Al realizar una compra por **transferencia bancaria**, se debe enviar un recordatorio para solicitar el comprobante de pago.

### Contexto

Se deben realizar **3 intentos** de recordatorio si todavía no se adjuntó el comprobante de la transferencia bancaria.

### Criterios de aceptación

- **Intento 1**  - Enviar a los **15 minutos** de la compra, si aún no se adjuntó el comprobante.
  - Asunto: “Gracias por tu compra!  Hace la transferencia y subi el comprobante! ”
- **Intento 2**  - Enviar a las **4 horas** de la compra, si aún no se adjuntó el comprobante.
  - Asunto: “Tu pedido está reservado, pero necesitamos el comprobante para avanzar”
- **Intento 3**  - Enviar a las **24 horas** de la compra, si aún no se adjuntó el comprobante.
  - Asunto: “Tu reserva está próxima a vencer: subí el comprobante de pago”
- Usar como referencia el siguiente template orientativo:

```
Hola {{customerName}},

Recibimos correctamente tu pedido <strong>#{{orderId}}</strong>, pero todavía no registramos el comprobante de la transferencia bancaria.

Para completar el pago, podés transferir a la siguiente cuenta:

<strong>Alias:</strong> libre.opcion <strong>CVU:</strong> 0000003100088833655790

Una vez realizada la transferencia, adjuntá el comprobante desde el siguiente enlace:

<a href="https://libreopcion.com.ar/compra/mis-compras/803172/comprobantes">Subir comprobante de pago</a>

<strong>Importante:</strong> Adjuntar el comprobante es indispensable para que podamos verificar tu pago y avanzar con el procesamiento del pedido.

Si ya realizaste la transferencia, simplemente subí el comprobante y nuestro equipo validará el pago a la brevedad.

Gracias por comprar en Libre Opción.
```

La ejecución podrá implementarse utilizando TASKv3 o sistema equivalente de colas y tareas programadas.

# Criterios de aceptación QA

1. El sistema debe enviar el recordatorio correspondiente únicamente para pedidos cuyo medio de pago sea transferencia bancaria y que no tengan comprobante adjunto.
2. El sistema debe enviar un máximo de 3 recordatorios por pedido:  - Intento 1: a los 15 minutos de realizada la compra.
  - Intento 2: a las 4 horas de realizada la compra.
  - Intento 3: a las 24 horas de realizada la compra.
3. Si el usuario adjunta el comprobante antes de alguno de los intentos programados, los recordatorios pendientes no deben enviarse.
4. Cada recordatorio enviado debe registrarse en el sistema para evitar reenvíos duplicados o ejecuciones fuera de término.
5. Cada correo debe contener:  - Número de pedido.
  - Alias y CVU para realizar la transferencia.
  - Enlace para adjuntar el comprobante.
  - Asunto correspondiente según el intento configurado.
6. El enlace para adjuntar comprobantes debe dirigir al detalle correcto de la compra correspondiente al pedido del usuario.
