# LIO-638 · API - CMS - Refactor -Recurso de palabras reactivas por categoria

[LIO-638 en Jira](https://bluinc.atlassian.net/browse/LIO-638)

## Descripción

### Resumen

Se agregará un repositorio para gestionar parámetros o keywords de palabras reactivas. Se permitirá abrir categorías en un modal para modificar o agregar palabras.

### Contexto

El objetivo es facilitar el acceso a las categorías de Matcheo. Los usuarios podrán editar palabras reactivas a través de un modal.

### Criterios de aceptación

- Acceso a un modal desde la sección de categorías.
- Listado de palabras cargadas en inputs de texto.
- Opción para modificar palabras y guardarlas.
- Opción para remover palabras.
- Opción para agregar nuevas palabras.
- Uso del wildcard “%” en todos los casos.
- Implementación de los siguientes verbos:  - ```
GET/DELETE/POST/PATCH {API_URL}/v1/categoriasPrediccion=MatchFree?categoryId&categoryId={categoryId}
```
  - ```
GET/DELETE/POST/PATCH {API_URL}/v1/categoriasPrediccion=Required&categoryId={categoryId}
```
  - ```
GET/DELETE/POST/PATCH {API_URL}/v1/categoriasPrediccion=Exclude&categoryId={categoryId}
```
- Generación de un objeto similar al siguiente:```json
[
  {
    "id": 5,
    "id_categoria": 203,
    "string_match": "%CÁMARA IP%"
  },
  {
    "id": 6,
    "id_categoria": 203,
    "string_match": "%CAMARA IP%"
  },
  {
    "id": 7,
    "id_categoria": 203,
    "string_match": "%CÁMARA EZVIZ%"
  },
  {
    "id": 8,
    "id_categoria": 203,
    "string_match": "%CAMARA EZVIZ%"
  }
]
```
