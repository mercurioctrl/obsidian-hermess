# Arquitectura — inventario

## Estructura general

Monorepo con dos subproyectos independientes, cada uno con su propio `.git` y CI/CD:

```
inventario/
├── inventario-web-app/app/   # Frontend SPA
└── ms-metadata/              # Backend API REST
```

## Frontend (inventario-web-app)

- **Patrón**: SPA (Single Page Application) con SSR deshabilitado en producción
- **Routing**: File-based en `pages/` (Nuxt convention). Rutas extendidas en `nuxt.config.js` con `router.extendRoutes` para agregar variante `/:find` de búsqueda
- **Auth**: `@nuxtjs/auth-next` con estrategia JWT local. Todas las rutas protegidas via middleware global en `nuxt.config.js`. Endpoints: `/auth/login`, `/auth/user`, `/auth/logout`
- **API Layer**: Centralizado en `plugins/api.js`, inyectado como `this.$api`. Axios apunta a `API_HOST`
- **Estado global**: Vuex en `store/` — principalmente paginación y definición de columnas de tablas. La carga de datos se hace en los componentes de página
- **UI**: Ant Design Vue v1, Less para estilos (`assets/ant/main.less`, `assets/less/colors.less`)
- **Filtros globales**: `formatNum` en `plugins/formats.js` — formato argentino (`.` miles, `,` decimales)

## Backend (ms-metadata)

- **Patrón**: FastAPI con entry point único en `main.py` (todas las rutas definidas ahí)
- **Auth**: JWT Bearer (`auth/auth_bearer.py`, `auth/auth_handler.py`). Token válido si tiene clave `usuario` en el payload
- **Base de datos**: SQL Server via pyodbc (raw SQL, sin ORM). Conexión por-request en `core/database/database.py:dbconnection()`. El cursor debe cerrarse manualmente
- **Controladores**: Organizados por dominio en `core/controllers/{dominio}/`, llamados directamente desde los handlers de `main.py`
- **Modelos**: Pydantic en `core/models/models.py` y `core/models/product.py`
- **CORS**: Lista de orígenes desde `ALLOWED_ORIGINS` env var (comma-separated)

### Dominios de controladores

| Dominio | Descripción |
|---------|-------------|
| `products/` | CRUD de ítems, creación de stock, scraping de atributos |
| `stocks/` | Gestión de stock, transferencias entre depósitos, seriales |
| `brands/`, `categories/` | Datos maestros |
| `prices/` | Precios y utilidades |
| `selldiscount/` | Reglas de descuento en venta |
| `kits/` | Kits/bundles (costo y utilidad incluidos) |
| `certificates/` | Certificados eléctricos con generación de etiquetas ZPL |
| `distributors/` | Gestión de distribuidores |
| `ia/` | Generación de descripciones y extracción de atributos via IA (OpenAI) |
| `scrappers/` | Scrapers: MercadoLibre, Newegg, código de barras (Scrapy) |
| `mapping/` | Importación desde Excel |
| `warehouses/` | Depósitos |
| `user/` | Usuarios |

## Deployment

- **Frontend**: PR a rama `gamma` → GitHub Actions → SSH → `deployInventory.sh` → pm2
- **Backend**: PR a rama `Gamma` → GitHub Actions → SSH → `git pull origin Gamma` → auto-PR a `Development`
