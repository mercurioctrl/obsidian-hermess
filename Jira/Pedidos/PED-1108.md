## Descripción

Actualmente en el sistema existen varios apartados donde se utiliza el concepto `ID_ALMACEN`. A partir de ahora este campo pasará a denominarse `stockWarehouseId`.

Se requiere **refactorizar el repositorio y el endpoint**:

```
GET {API_URL}/v1/items
```

para que en cada registro de ítem se incluya el parámetro `stockWarehouseId`.

En caso de que un mismo ítem pertenezca a más de un `stockWarehouseId`, el endpoint deberá devolver **un registro por cada combinación** (la duplicación es correcta), incluyendo el valor de `stockWarehouseId`. Esto permitirá que el usuario seleccione el almacén correspondiente.

### **Criterios de aceptación**

- El endpoint `/v1/items` debe devolver el nuevo campo `stockWarehouseId` en cada registro.


- No se debe afectar la velocidad del repositorio y para eso debe cumplirse sin excepción la regla de los 2s


- Si un ítem pertenece a múltiples almacenes, debe devolverse un registro por cada `stockWarehouseId` asociado.


- La duplicación de ítems con distinto `stockWarehouseId` es intencional y se considera válida.


- El resto de la estructura de respuesta no debe verse afectada.
