## Descripción

Actualmente, al generar un comprobante manual mediante el endpoint `makeVoucherManual`, aunque se envíe un valor en el parámetro `cotización`, el sistema siempre toma automáticamente el tipo de cambio del día.

```
POST {API_URL}/v1/makeVoucherManual
```



**Requerimiento:**
Realizar un refactor para que, en caso de que se reciba el parámetro `cotización` en la request, el sistema lo utilice como tipo de cambio en lugar del valor por defecto del día.

```
curl 'https://gamma.api.orders.lio.red/v1/makeVoucherManual' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  --data-raw '{
    "clientId": 26806,
    "voucherType": "FACTURA",
    "cotizacion": "1000", <----- 
    "trade": [
      {
        "internalId": 5775,
        "units": 1,
        "price": "1",
        "ivaTax": 21,
        "internalTax": 0,
        "description": "rwwar"
      }
    ],
    "impactCurrentAccount": false
  }'

```

**Criterios de aceptación:**

- Si el parámetro `cotizacion` **está presente** en la request, se debe usar ese valor como tipo de cambio.


- Si el parámetro `cotizacion` **no está presente**, se debe continuar utilizando el tipo de cambio del día (comportamiento actual).
