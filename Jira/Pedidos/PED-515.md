## Descripción

Crearemos un SyncUp cuya finalidad es correrlo en un tarea.

```
GET {{API_URL}}/v1/syncUp/purchaseConfirmation?token={token en .env}
```

Lo que hace es agregarle una palabra clave en el parámetro `secret_key` y al mismo tiempo enviarla por correo como se hace en la API de NB al cliente. Esto lo hace en base al recurso del servicio de envios 

```
GET {{API_URL}}/shipping/getPrivateKey
```

Para no hacer el envío varias veces lo que haremos sera registrar de alguna forma, posiblemente en otra tabla, los envíos que ya realizamos. 

Aprovecharemos este mismo momento para enviar completo el correo que enviamos con la API cuando hacen un pedido a modo de confirmación de la compra.
