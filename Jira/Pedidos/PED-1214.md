## Descripción

**Qué hay que hacer**

- Insertar un movimiento (gasto real) imputado a `FundId` y `ActionId`, opcionalmente `AllocationId`.


- Calcular y guardar `AmountFinalFundCurrency` usando `FxRateToFund` cuando corresponda.


- Actualizar stats:

- `Marketing_FundStats.SpentTotal += AmountFinalFundCurrency`


- `Marketing_ActionStats.SpentTotal += AmountFinalFundCurrency`


- (opcional recomendado) `Marketing_AllocationStats.SpentTotal += AmountFinalFundCurrency` si hay `AllocationId`.





**Endpoint**

```
POST /v1/marketing/movements
```

**Ejemplo request (payload)**

```
{
  "fundId": 101,
  "actionId": 501,
  "allocationId": 9001,
  "conceptShort": "Meta Ads - campaña Hot Sale",
  "conceptLong": "Factura 0004-00001234 / pauta del 11-05",
  "originalCurrency": "ARS",
  "originalAmount": 300000.00,
  "fxRateToFund": 0.00125,
  "occurredAt": "2026-05-11T10:30:00-03:00"
}

```

**Ejemplo response (201)**

```
{
  "id": 70001,
  "fundId": 101,
  "actionId": 501,
  "allocationId": 9001,
  "conceptShort": "Meta Ads - campaña Hot Sale",
  "originalCurrency": "ARS",
  "originalAmount": 300000.00,
  "fxRateToFund": 0.00125,
  "amountFinalFundCurrency": 375.00,
  "occurredAt": "2026-05-11T10:30:00-03:00",
  "createdAt": "2026-01-05T06:20:00-03:00"
}

```

**SQL Server**

- `NB_WEB.dbo.Marketing_Movements`


- `NB_WEB.dbo.Marketing_FundStats`


- `NB_WEB.dbo.Marketing_ActionStats`


- (opcional recomendado) `NB_WEB.dbo.Marketing_AllocationStats`



**Criterios de aceptación**

- Requiere `FundId`, `ActionId`, `ConceptShort`, `OriginalAmount>0`, `OccurredAt`, `OriginalCurrency`.


- Si `OriginalCurrency == Fund.Currency`, fuerza `FxRateToFund=1` y `AmountFinalFundCurrency=OriginalAmount`.


- Si difiere, calcula `AmountFinalFundCurrency = ROUND(OriginalAmount * FxRateToFund, 2)`.


- No permite dejar el fondo en negativo (`SpentTotal + AmountFinalFundCurrency <= AmountOriginal`).


- Si viene `AllocationId`, valida que corresponda a ese `FundId+ActionId` y que no se exceda el asignado.



**SQL Server (tablas)**

### `NB_WEB.dbo.Marketing_Movements`

- `Id` int IDENTITY PK


- `FundId` int NOT NULL FK → `Marketing_Funds(Id)`


- `ActionId` int NOT NULL FK → `Marketing_Actions(Id)`


- `AllocationId` int NULL FK → `Marketing_Allocations(Id)` *(recomendado para control de presupuesto)*


- `ConceptShort` varchar(120) NOT NULL


- `ConceptLong` varchar(800) NULL


- `OriginalCurrency` char(3) NOT NULL


- `OriginalAmount` decimal(18,2) NOT NULL


- `FxRateToFund` decimal(18,6) NOT NULL


- `AmountFinalFundCurrency` decimal(18,2) NOT NULL


- `OccurredAt` datetime2 NOT NULL


- `CreatedAt` datetime2 NOT NULL DEFAULT sysdatetime()


- `CreatedByUserId` int NULL



Índices:

- `IX_MMov_FundId_OccurredAt` (`FundId`,`OccurredAt`)


- `IX_MMov_ActionId_OccurredAt` (`ActionId`,`OccurredAt`)


- `IX_MMov_AllocationId` (`AllocationId`)
