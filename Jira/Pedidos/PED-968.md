## Descripción

Actualmente, la validación de estados para desliquidar una orden está implementada de forma rígida en el código. Para mejorar la flexibilidad y facilitar ajustes sin necesidad de cambios en permisos específicos, se incorporará un parámetro configurable en las variables de entorno (`.env`).

Este parámetro permitirá definir qué estados pueden habilitar la desliquidación, evitando modificaciones innecesarias en el código cada vez que se requiera un ajuste.

#### **Alcance**

Se refactorizará el recurso:


```
 PATCH {{API_URL}}/v1/openSale
```

Se introducirá un nuevo parámetro en las variables de entorno con los IDs de los estados permitidos, tomando como referencia la tabla **[NEW_BYTES].[dbo].[MS_STATUS_REMITO]**.

Inicialmente, se configurará solo el estado **1 (Pendientes a Autorizar)**, con posibilidad de agregar otros en el futuro según necesidades operativas.

#### **Objetivo**

✅ Mayor flexibilidad en la gestión de estados.
✅ Permitir ajustes rápidos sin afectar la lógica de permisos generales.



Var en .env

```
# Estados permitidos para desliquidar un pedido (valores separados por coma) ej: 1,2,3,4
REMOVABLE_LIQUIDATION_STATUS=1
```
