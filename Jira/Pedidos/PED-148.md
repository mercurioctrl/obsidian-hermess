## Descripción

**Consultar información de un usuario**

Objetivo: Desarrollar un recurso API que permita obtener la información de un usuario existente a partir de su identificador.

**Especificación del recurso:**

```
GET {API_URL}/v1/user/{id_cliente} 
```

**Respuesta esperada:**

```
{
  "username": "Nombre de usuario",
  "showName": "Nombre a mostrar del usuario",
  "disabled": "Indica si el usuario está activado (true/false)",
  "mail": "Correo electrónico del usuario"
}

```

**Notas:**

- Esta implementación se basa en la tabla de usuarios y sólo muestra la cuenta principal, es decir, aquellas donde `roleId = 1` y `subcuenta` es nula.


- No se debe incluir la contraseña en la respuesta por motivos de seguridad.



**Consulta SQL de referencia:**

```
SELECT username, nomPmostrar, desactivada, UserEmail
FROM [NB_WEB].[dbo].[usuarios_nb]
WHERE id_cliente = ? AND roleId = 1 AND subcuenta IS NULL;
```
