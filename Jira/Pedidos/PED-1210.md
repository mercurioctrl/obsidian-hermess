## Descripción

**Qué hay que hacer**

- Listar fondos con totales desde `Marketing_FundStats` y saldos calculados (sin vistas).



```
GET /v1/marketing/funds?brandId={brandId}&currencyId={1|2}&activeOnly={0|1}&currentPage=1&itemsPerPage=20
```

**Ejemplo request (llamada)**

```
GET /v1/marketing/funds?brandId=12&currencyId=1&activeOnly=1&currentPage=1&itemsPerPage=20
```

**Ejemplo response (200)**

```
{
  "pagination": {
      "total": 1293,
      "current": 3,
      "pageSize": 15
  }
  "list": [
    {
      "id": 101,
      "brandId": 12,
      "name": "ASUS - Q1 2026 - USD 5000",
      "currencyId": "USD",
      "amountOriginal": 5000.00,
      "allocatedTotal": 1200.00,
      "spentTotal": 600.00,
      "availableToAllocate": 3800.00,
      "availableCash": 4400.00,
      "expiresAt": "2026-03-31T23:59:59-03:00",
      "createdAt": "2026-01-05T06:12:00-03:00"
    }
  ]
}

```

**SQL Server**

- `Marketing_Funds`


- `Marketing_FundStats`



**Criterios de aceptación**

- Devuelve paginado y ordenado por `CreatedAt DESC`.


- Filtros combinables: `brandId`, `currencyId`, `activeOnly`.


- Incluye `AllocatedTotal`, `SpentTotal`, `AvailableToAllocate`, `AvailableCash`.


- Si no viene definido cualquiera de los filtros (`brandId`, `activeOnly`, `currencyId`), simplemente muestro todos los resultados para el filtro.


- Si stats faltan (inconsistencia), no rompe: devuelve 0 + `hasInconsistency=true` (o equivalente).
