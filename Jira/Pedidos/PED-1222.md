## Descripción

UI para asignar monto desde un fondo a una acción (presupuesto máximo). Debe mostrar el disponible del fondo para reducir intentos inválidos.

**API**

```
POST /v1/marketing/allocations
```

- (para mostrar contexto del fondo) usa datos ya cargados desde `GET /funds` o refresca si hace falta.



**Ejemplo payload**

```
{ "fundId": 101, "actionId": 501, "amountAllocated": 1200 }
```

**Criterios de aceptación**

- Permite seleccionar `fundId` y `actionId` (o precarga si viene desde un contexto).


- Muestra `availableToAllocate` del fondo y valida `amountAllocated > 0` y `<= availableToAllocate`.


- Al éxito, refresca `GET /v1/marketing/funds?brandId=...` y `GET /v1/marketing/actions`.


- Si el backend rechaza por límites, muestra error claro (“excede disponible/ya existe asignación”).


- Cierra modal/pantalla solo si el POST fue exitoso.
