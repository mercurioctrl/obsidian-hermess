## Descripción

La idea es crear un recurso que nos permita hacer, que un movimiento determinado que es una devolución de dinero, aparezca en la wallet (genera hmac con la misma clave), pero a través de la intervención manual de un comercial

```
POST {API_URL}/v1/currentAccount/{clientId}/refundToWallet
```

```json
{
  "id": 123456 --> id de movimiento cta cte
}
```



**¿que controlaremos?**

- Que el movement_id exista.



- Que el `TR_CODIGO` de la tabla `MC_CCORRIENTES_MOVIMIENTOS` sea `30`


- Que no se hiciera anteriormente para este mismo movimiento el `refundToWallet`
