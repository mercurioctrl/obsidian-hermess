## Descripción

Al ejecutar el endpoint 

```
POST {API_URL}/v1/closeSale
```

se deben actualizar los procesos vinculados a la generación de remitos para que incorporen el campo `stockWarehouseId`.
Este campo permitirá conservar la trazabilidad del depósito de origen desde donde salió el stock del ítem vendido.

---

### **Detalles técnicos**

Se deberá agregar la columna `stockWarehouseId` en las siguientes tablas:

- `[NEW_BYTES].[dbo].[MS_REMITO_DETALLE_GANANCIA_ENLACE].stockWarehouseId`


- `[NEW_BYTES].[dbo].[MS_REMITO_DETALLE].stockWarehouseId`



El valor debe ser el mismo que se hereda desde:

- `[NewBytes_DBF].[dbo].[albclil].stockWarehouseId`



Esto garantiza que, al momento de cerrar la venta (`closeSale`), el detalle de cada ítem refleje correctamente el depósito desde el cual se descontó el stock.

Si es necesario modificar estructuras de tablas para admitir la nueva clave única id_articulo + pedido + deposito se debe dejar la modificación en la historia
