## Descripción

En este ultimo caso nos encargaremos de guardar `[NEW_BYTES].[dbo].[MS_REMITO_CABECERA].IMPPERCEP` como el monto total de todas mis percepciones sumadas (CABA y ARBA) de modo tal que no sea disruptivo y tengamos que refactorizar todos los sistemas.

Pero ademas, nos encargaremos tambien de guardarlas por separado dado que al momento de realizar el comprobante fiscal, si debemos poder saber cual es cual y sus monto para poder informarlo en el comprobante.

Refactorizaremos el recurso 

```
POST {API_URL}/v1/closeSale
```

Para esto haremos los siguientes pasos

- Al momento de agregar la percepción en `[NEW_BYTES].[dbo].[MS_REMITO_CABECERA].IMPPERCEP` nos aseguraremos que ambas estén sumadas en la linea de lo que venimos haciendo y como montoa nominales (**App\Services\Liquidate\CreateOrder::buildHeader**)


- Se debe modificar `[NEW_BYTES].[dbo].[MS_REMITO_CABECERA].TOTALREMITO` para agregarle la nueva percepción (como siempre solo fomra parte del total final)


- Crearemos una nueva tabla `[NEW_BYTES].[dbo].[MS_REMITO_PERCEPCIONES]` con el único objetivo de guardar las percepciones mas ordenadas para este objetivo (Esto lo guardamos en una tabla a parte, porque ademas de que existe la posibilidad de que se agregue para mas provincias, seguro construiremos informes específicos de ingresos brutos en base a esta tabla)

Columnas: 

```sql
	[SUCURSAL_REMITO] [nvarchar](4) NOT NULL,
	[REMITO_FP] [nvarchar](20) NOT NULL,
	[IMPPERCEP_CABA] [float] NULL,
	[IMPPERCEP_ARBA] [float] NULL,
	[IMPPERCEP_CABA_P] [float] NULL, (lo mismo pero porcentual, en vez de nominal)
	[IMPPERCEP_ARBA_P] [float] NULL, (lo mismo pero porcentual, en vez de nominal)
	[ID_IMPUESTO] [int] <-- Esto es segun la tabla  [NewBytes_DBF].[dbo].[FP_IMPUESTOS] (5 y 10 respectivamente)
```
