## Descripción

Permitir convertir un artículo ya existente en la tabla `[NewBytes_DBF].[dbo].[articulo]` en un **kit**, marcando el flag correspondiente.

### Endpoint

```
POST {API_URL}/itemsKits
```

### Request (ejemplo)

```
{
  "itemId": 12345
}
```

- Validar que `itemId` exista en `[NewBytes_DBF].[dbo].[articulo]`.


- Validar que **todavía no sea kit para evitar kits circulares** (`kit = 0`).


- Marcar el artículo como kit:

```
UPDATE [NewBytes_DBF].[dbo].[articulo]
SET kit = 1
WHERE itemId = @itemId;
```



- Esta operación **no crea componentes** ni modifica `ncosteprom`.

- El costo del kit se recalculará recién cuando se configuren componentes vía `PATCH` / `DELETE`.





### Respuesta (ejemplo OK)

```
{
  "success": true,
  "message": "El artículo fue marcado como kit",
  "itemId": 12345
}

```

### Criterios de aceptación

- Dado un `itemId` existente y no kit, al llamar `POST /itemsKits`:

- Se actualiza `[articulo].kit = 1`.


- Devuelve `success = true`.




- Al intentar marcar como kit un `itemId` inexistente → `404`.


- Al intentar marcar como kit un artículo que ya es kit → `409` o error equivalente.
