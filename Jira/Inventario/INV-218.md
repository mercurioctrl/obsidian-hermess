## Descripción

Crear un endpoint de solo lectura que devuelva el **historial de movimientos de stock** (compras, ventas, transferencias entre depósitos y movimientos entre buckets) para un `itemId` y un `warehouseId` (u otros filtros), leyendo de `[NB_WEB].[dbo].[registro_stock]`.
El recurso debe incluir:

- Fecha/hora del movimiento


- Documento (comprobante o referencia)


- Agente (usuario/sistema)


- Fichero/Origen (servicio o script que generó el movimiento)


- Cantidad del movimiento (negativo = salida, positivo = entrada)


- Stock previo y posterior (del **bucket principal afectado**)


- Snapshot **previo** (antes del movimiento) de todos los buckets:

- `nstock_lo` → **STKLO previo**


- `nstock` → **STK previo**


- `nstock_en_cola` → **COLA previo**


- `nstock_d1` → **STK0 previo**


- `nstock_reserva_pedidos` → **PED previo**


- `nstock_lo_reserva_pedidos` → **PEDLO previo**


- `nstock_postventa` → **POSTV previo**




- Info extra (IP, OS, navegador, user_agent, etc. cuando exista)



> **Nota de consistencia:** Este recurso **no modifica** datos. Solo lee y ordena resultados para la UI con paginación y filtros.


```
GET {API_URL}/v1/stock-history?itemId={itemId}&warehouseId={warehouseId}&between=05-11-2025_12-11-2025&currentPage=1&itemsPerPage=300
```

## Criterios de aceptación

-  `GET /v1/stock-history` devuelve el historial filtrado y paginado 


-  Orden por defecto `fecha DESC`, con `currentPage` y `itemsPerPage`.


-  Filtros por `itemId` (requerido), `warehouseId`, rango de fechas, `document`, `agent`, `source`, `movementType`, `bucket`.


-  Maneja zona horaria `America/Argentina/Buenos_Aires` en la serialización de fecha/hora.


-  Respuestas de error claras ante parámetros inválidos.


-  Performance adecuada con índices propuestos y `OFFSET/FETCH`.
