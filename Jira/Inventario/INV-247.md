## Descripción

Se implementará un nuevo recurso que permita **inspeccionar en detalle** el origen de los valores que se muestran en el repositorio de *stock general*.
El objetivo es tener **transparencia y trazabilidad completa** de cada cantidad, mostrando **qué documentos comerciales** generan esos números (notas, pedidos, ingresos, devoluciones, etc.).

Este recurso funcionará como un **repositorio unificado de auditoría** para analizar cualquier parámetro relevante del stock.

---

## **Nuevo endpoint**

```
GET {API_URL}/itemsStocks/examine?between=12-11-2025_13-11-2025&itemId=3434&warehouseId=2&type={type}

```

### **Parámetros**

- **between**: rango de fechas para filtrar documentos.


- **itemId**: ID del artículo a inspeccionar.


- **warehouseId**: depósito.


- **type**: tipo de elemento a auditar (ver listado abajo).



---

## **Tipos inspeccionables (type)**

Por ahora se podrán inspeccionar los siguientes orígenes de datos:

- **salesReserved**


- **aftersalesCreditNote**


- **creditNoteReturn**


- **inProviderOrderInbound**


- **inProviderOrder**


- **stockInOrders**



Cada tipo consultará su repositorio correspondiente mediante una query específica.

---

## **Respuesta del recurso**

Siempre devolverá un objeto estructurado así:

```
{
  "data": [
    {
      "date": "01-01-01 01:01",
      "linkedDocument": "0002-3334532",
      "amount": 3,

      "clientId": 2333,
      "clientName": "Nombre del cliente",

      "providerId": null,
      "providerName": null
    }
  ],
  "pagination": {
    "total": 3213,
    "pageSize": 50,
    "current": 1
  }
}

```

**Reglas importantes:**

- Si solo aplica **cliente**, los campos de proveedor van en *null*.


- Si solo aplica **proveedor**, los campos de cliente van en *null*.


- La paginación debe estar siempre presente.

# **Repositorios / Queries por tipo**

---

### **1. salesReserved**

```
SELECT
    albclit.id_nroremcli_enc AS linkedDocument,
    cnomcli AS clientName,
    ncanent AS amount,
    CONVERT(varchar, dfecalb, 20) AS date
FROM NewBytes_DBF.dbo.albclil
INNER JOIN NewBytes_DBF.dbo.albclit
    ON albclil.ID_NROREMCLI_ENC = albclit.ID_NROREMCLI_ENC
INNER JOIN NewBytes_DBF.dbo.articulo
    ON articulo.cRef = albclil.cref
INNER JOIN NewBytes_DBF.dbo.clientes
    ON clientes.ccodcli = albclit.ccodcli
WHERE albclil.ID_Articulo = @itemId
  AND albclit.ntipoalb > 1
  AND albclil.ID_ALMACEN = @warehouseId
  {date}
ORDER BY albclit.dfecalb DESC

```

### **4. inProviderOrderInbound**

```
SELECT
    albprot.nnumalb AS linkedDocument,
    FP_Proveedores.cnompro AS providerName,
    FP_Proveedores.ID_PROVEEDOR AS providerId,
    albprol.ncanent AS amount,
    CONVERT(varchar, albprot.dfecalb, 103) AS date
FROM NewBytes_DBF.dbo.albprol
LEFT JOIN NewBytes_DBF.dbo.albprot
    ON albprol.nnumalb = albprot.nnumalb
LEFT JOIN NewBytes_DBF.dbo.articulo
    ON articulo.cRef = albprol.cref
LEFT JOIN NewBytes_DBF.dbo.FP_Proveedores
    ON FP_Proveedores.ccodpro = albprot.ccodpro
LEFT JOIN NewBytes_DBF.dbo.PedProT
    ON albprot.nnumped = PedProT.nNumPed
WHERE albprol.ID_Articulo = @itemId
  AND albprot.ntipoalb > 1
  AND albprol.ID_ALMACEN = @warehouseId
  {date}
ORDER BY albprot.nnumped DESC

```

### **5. inProviderOrder**

```
SELECT
    PedProT.nNumPed AS linkedDocument,
    proveedo.cnompro AS providerName,
    proveedo.ID_PROVEEDOR AS providerId,
    pedprol.nCanEnt AS amount,
    CONVERT(varchar, PedProT.dFecPed, 103) AS date
FROM NewBytes_DBF.dbo.pedprol
LEFT JOIN NewBytes_DBF.dbo.PedProT
    ON pedprol.nNumPed = PedProT.nNumPed
INNER JOIN NewBytes_DBF.dbo.articulo
    ON articulo.cRef = pedprol.cref
INNER JOIN NewBytes_DBF.dbo.proveedo
    ON proveedo.ccodpro = PedProT.ccodpro
WHERE pedprol.ID_Articulo = @itemId
  AND pedprol.ID_ALMACEN = @warehouseId
  AND PedProT.cEstado = 'p'
  {date}
ORDER BY PedProT.nNumPed DESC

```

### **6. stockInOrders**

```
SELECT
    CONVERT(VARCHAR, A.dfecped, 20) AS date,
    A.cnumsuc,
    A.cnumsuc + A.cnumped AS linkedDocument,
    D.cnomcli AS providerName,
    D.providerId AS providerId,
    B.ncanped AS amount
FROM NewBytes_DBF.dbo.pedclit A
LEFT JOIN NewBytes_DBF.dbo.pedclil B
    ON A.cnumsuc = B.cnumsuc AND A.cnumped = B.cnumped
LEFT JOIN NewBytes_DBF.dbo.articulo C
    ON C.cRef = B.cref
LEFT JOIN NewBytes_DBF.dbo.clientes D
    ON D.ccodcli = A.ccodcli
LEFT JOIN NewBytes_DBF.dbo.agentes E
    ON E.ccodage = A.ccodage
WHERE B.ID_Articulo = @itemId
  AND B.ID_ALMACEN = @warehouseId
  AND A.cobserv IN ('INTERNO', 'DESCARGADO', 'PEDIDO LIBRE OPCION')
  AND A.cestado = 'P'
  AND B.ncanped > 0
  AND lanula <> 1
  {date}
GROUP BY
    A.dfecped, B.ncanent, A.cnumsuc, A.cnumped,
    D.cnomcli, B.ncanped, E.cnbrage, E.capeage
```
