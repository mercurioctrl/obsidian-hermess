## Descripción

Agregaremos un nuevo parámetro como obligatorio para ingresar el parámetro` [NewBytes_DBF].[dbo].[articulo].packagePerUnit` (cuantos bultos ocupa, o bien cuantos viene en una caja)

Así como lo hicimos con las medidas, donde las pide como obligatorias al intentar serializar por primera vez solo si el parámetro `VALIDATE_MEASURES=1` pero en este caso introduciremos el parámetro `VALIDATE_PACKAGEPERUNIT` en nuestras variables de entorno

```
POST {API_URL}/v1/providersOrders/{order}/serials/{item}
```

Es decir que solo lo mandaremos como obligatorio si el mismo esta en `VALIDATE_PACKAGEPERUNIT=1`

```
{
    "error": "El producto no tiene medidas o serials",
    "errorFields": [
        "packagePerUnit" <-- Se agrega
        "weightAverage",
        "widthAverage",
        "highAverage",
        "lengthAverage",
        "ean",
        "upc",
        "isbn",
        "gtin"
    ],
    "data": {
        "packagePerUnit": null, <-- Se agrega
        "ean": null,
        "upc": null,
        "gtin": null,
        "isbn": null,
```

IMPORTANTE: Como la pregunta que haremos es cuantas unidades vienen en la caja para poder deducir mas o menos cuantas unidades forman un “bulto” entonces lo que haremos sera una división ya que para nosotros el valor `packagePerUnit` es la cantidad de bultos.

Es decir por ejemplo:

Si la respuesta es que vienen **10 por caja **

**Nuestro valor en **`packagePerUnit`** es 0.1 **para poder seguir usando el mismo criterio.

**Si vienen 20 por caja nuestro valor sera 0.5**

Formalmente seria 1/cantidad ingresada = `packagePerUnit`
