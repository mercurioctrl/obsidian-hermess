## Descripción

Se trata de un refactor necesario para poder mostrar que cliente es en realidad, cuando el cliente en el registro principal es el de libre opción.

Primer paso, crearemos en el .env una variable de entorno del tipo CLIENTE_INTERNO_A En este cuando el cliente (en este caso 32103).

Eso nos sirve para saber cuando el cliente que estamos asignados actualmente como clientId es igual a CLIENTE_INTERNO_A entonces debemos buscar sus datos de manera secundaria.

Para eso buscaremos el cliente vinculado LO.dbo.usuarios

    "clientId": "019227", // lo cambiamos por el cliente nuevo asignado

    "userId": 7463, //mantenemos el de libre opcion

  },
