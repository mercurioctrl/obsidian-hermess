## Descripción

Este recurso es la base para crear cualquier comprobante o voucher de cualquier tipo, y posee los parámetros mínimos generales para realizar uno.

```
POST {API_URL}/v2/CreateVoucher
```

```
{
  voucherTypeId: 2, // sale de https://lioteam.atlassian.net/browse/POS-134
  clientId: 45455,
  trade:[
          {
          units:1,
          price: 23.55,
          ivaTax:21,
          internalId: 324 //opcional
          description: 'Esta es una descripcion opcional' //opcional
          }
          {
          units:11,
          price: 223.55,
          ivaTax:10.5,
          }
          {
          units:3,
          price: 2.35,
         ivaTax:21,
          }
  ]
}
```

Básicamente se compone de parámetros para setear el voucher que voy a realizar como ser:

- **voucherTypeId**: Que es el tipo de comprobante definido es sus mayores grupos (nota de crédito, factura, nota de débito) [https://lioteam.atlassian.net/browse/POS-134](https://lioteam.atlassian.net/browse/POS-134)


- **clientId** : Que es el numero de cliente de la tabla clientes. Con eso obtendremos internamente todos los datos necesarios para confeccionar la base de nuestro comprobante con la misma lógica que [link](https://lioteam.atlassian.net/browse/POS-133) 


- **trade**: Es un objeto confeccionado sobre la base de los productos que quiero realizar el comprobante.



**Mas sobre el objeto trade**

- Units: se trata de la cantidad de unidades


- Price: Es el precio unitario sin impuestos.


- ivaTax: Es el impuesto al iva del producto.


- internalId: Es el id de un item en la tabla newbytes_dbf.dbo.articulo (Opcional para cuando el objeto se construye desde un pedido u otro documento)


- description: Es una descripción para el producto. (Opcional)



En base a este objeto construiremos los totales con iva, los totales sin iva y demas montos que requiere el objeto de afip para crear el voucher.

**ADEMAS**

Absolutamente todos los comprobantes deben quedar registrados en las tablas

`FP_FactWebCliDetalle` y `FP_FactWebCliEncabezado` respectivamente con el numero de CAE y su fecha de vencimiento.

Investigar estas tabalas para ver que valores son obligatorios.
