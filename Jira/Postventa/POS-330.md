## Descripción

Se agregará un nuevo **accionable** al menú contextual (botón derecho) sobre las órdenes de postventa finalizadas, que permitirá invocar el recurso 


```
POST /v1/afterSales/{id}/finalized/createReturnOrder
```


Antes de ejecutar la acción, se mostrará una ventana de confirmación al usuario.

---

**Funcionalidad esperada:**

- En el menú contextual de órdenes de postventa finalizadas, agregar la opción:
➤ **"Genear orden de Envio"**


- Al hacer clic en esta opción, mostrar un **mensaje de confirmación**:

*“Se creará una orden en el sistema de pedidos para que el vendedor continúe el procedimiento de enviar la mercadería al cliente. También se notificará al cliente por correo electrónico. ¿Deseás continuar?”*


- Si el usuario acepta, ejecutar el POST al backend.


- Si el usuario cancela, no se ejecuta ninguna acción.


- Mostrar feedback visual del resultado (éxito o error).



---

**Criterios de aceptación:**

- ✅ El nuevo ítem del menú contextual debe aparecer **solo para órdenes finalizadas**.


- ✅ El mensaje de confirmación debe mostrar claramente las consecuencias de la acción.


- ✅ Si se acepta, debe ejecutarse el endpoint correctamente y mostrar mensaje de éxito o error.


- ✅ La acción debe mostrar el mensaje que envia el backend de rechazo cuando la postventa no es del tipo permitido (“NO FALLO” o “Cambio”).


- ✅ Se debe prevenir múltiples envíos accidentales (ej. deshabilitar botón mientras se procesa).
