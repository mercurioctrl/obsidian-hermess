## Descripción

```
PATCH {API_URL}/v1/providerOrder/{providerOrdeId}/head
```

```
{
  "trackingNumber": "AA9330SDFDKFF" <-- SE AGREGA
  "arrivalDate": {dateTime} <-- SE AGREGA
}
```

## Parametros en la tabal que afectaremos

```
  [NewBytes_DBF].[dbo].[PedProT].trackingNumber

  [NewBytes_DBF].[dbo].[PedProT].arrivalDate
```

Criterios de Aceptación

- El recurso `PATCH {API_URL}/v1/providerOrder/{providerOrderId}/head` debe permitir actualizar `trackingNumber` y `arrivalDate`.


- Los datos deben persistirse en los campos correspondientes de la tabla `[NewBytes_DBF].[dbo].[PedProT]`:

- `trackingNumber`


- `arrivalDate`




- La implementación no debe afectar ni alterar el comportamiento de otros campos de la orden de compra.


- `arrivalDate` debe manejarse como un `datetime` válido y ser consistente con los formatos actuales del sistema.


- Se debe permitir que `trackingNumber` y `arrivalDate` sean nulos si no hay valores definidos.
