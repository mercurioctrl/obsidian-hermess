## Descripción

Así como pasa en compras, existe un proceso mediante el cual la “orden de compra” pasa a nuestro inventario (suma stock) y este proceso es cuando se genera un “ingreso” a partir de una orden de compra.

```
POST {API_URL}/v1/makeProviderOrderInbound
```

```
[
  {
    "orderNumber": 11082,
    "items": [
            {
                "id": 111770,
                "title": "ACCESORIOS GENERICO CABLE MONITOR VGA",
                "sku": "CABLE VGA",
                "price": {
                    "value": 100,
                    "iva": 21,
                    "finalPrice": 121
                },
                "warranty": "-",
                "amount": 22,

                "taxPosition": {

                    "dii": 0,

                    "ivaimp": 33

  }

```

**1 - Verificamos que tengo suficiente stock: ** Para saber si aun me queda mercaderia para pasar, tengo que validar que aun es posible hacerlo.
Para esto es necesario que para todos los productos que estoy queriendo ingresar aun tenga disponibilidad para hacerlo.  Para esto, los pendientes de ingreso tienen que ser mayores o iguales que cero.

```
pendientes de ingreso = amount - amountEntered - partialIncome
```

De esta forma podemos usar el parámetro `partialIncome` para ir ingresando parcialmente el stock de la orden.



**2 - Generar cabecera del ingreso: ** Lo que hace primero es crear la cabecera para el ingreso, para esto agregaremos una nueva fila en 

```
NB_DBF.dbo.albprot
```

Nos aseguramos de incrementar `nnumalb` manteniendo el formato "string" orginal. Si no existe un indice autonumerico o `id`

También de marcar todas las columnas como proveedor (ccodpro), divisa (ccoddiv), etc. Pero tambien agregaremos sus indices numéricos (id_divisa,id_proveedor) 



**3 - Generar detalle del ingreso: ** basándonos en el detalle de la “orden de compra” generaremos el del ingreso

```
NB_DBF.dbo.albprol
```

Simplemente tomamos los items de pedprol para pasarlos a nuestra nueva columna

**4- Agregamos el stock al inventario**

```
[NewBytes_DBF].[dbo].[stocks].nstock = nstock + partialIncome
```

**5- Marcamos la “ultimo ingreso” para cada item**

```
UPDATE NB_DBF.dbo.articulo SET ULTIMO_INGRESO = GETDATE() WHERE cref =?
```



**6- Marcamos el ingreso para cada item, según la cantidad  **

En la linea de lo que hicimos en el paso 1 debemos modificar el `amountEntered` de la orden sumándole lo que acabamos de ingresar, osea en la tabla

```
[NewBytes_DBF].[dbo].[pedprol].nCanEnt
```

Recordemos que `nCanEnt` nunca puede ser mayor a `nCanped`
