## Descripción

```
{{API_URL}}/v1/auth/user
{
    "id": 62626,
    "codeFP": 46930,
    "username": "mmoreno",
    "email": "mmoreno@libreoopcion.com",
    "codeAgent": 60,
    "showName": "marbe",
    "usuIdentification": "",
    "compras": true,
    "pm": true,
    "companyCode": 0, <-------------------------- AQUÍ
    "unlockedCompanyFilter": true
}
```

Actualmente, cuando el usuario no tiene un `companyCode` asignado, el recurso retorna el el valor `'0'`.
 Se requiere refactorizar este comportamiento para que, en ausencia de `companyCode`, el valor sea `null`, manteniendo consistencia con el modelo de datos y evitando valores inválidos.
