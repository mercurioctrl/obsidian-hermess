## Descripción

Agregar en el modal de detalle de Orden de Compra la visualización del **costo sugerido (**`suggestedCost`**)** por ítem, y permitir su **edición manual** mediante un accionable directo sobre cada producto.

[attachment]
---

### Alcance

- Al abrir el **modal de la orden**:

- Mostrar el campo `suggestedCost` cuando esté disponible.


- Si no existe, mostrar vacío o `-`.




- Sobre cada ítem:

- Incluir un **accionable rápido** (ej: click derecho / menú contextual / botón editar) para:

- Cargar o modificar el `suggestedCost`.






- Al confirmar el cambio:

- Enviar `PATCH /v1/providerOrder/{purchaseOrderId}` con:

```
{
  "item": {
    "id": <itemId>,
    "suggestedCost": <valor>
  }
}

```




- Refrescar el valor en pantalla tras respuesta exitosa (sin recargar el modal).



---

### UI / UX

- Mostrar el costo sugerido como una **columna adicional** o un **badge editable** dentro del listado de ítems.


- El input debe validar:

- Números positivos.


- Dos decimales máximo.





---

### Criterios de aceptación

- El modal muestra correctamente `suggestedCost` por ítem cuando está disponible.


- El usuario puede editarlo desde el item sin abandonar la pantalla.


- El cambio persiste en backend y se refleja inmediatamente en el UI.


- No se impacta la operatoria normal de carga de órdenes.
