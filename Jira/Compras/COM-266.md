## Descripción

Incorporar un recurso para **dar de alta forwarders** como maestro logístico básico, asociado a una empresa (`companyCode`), reutilizable en compras, importaciones y operaciones.

```
POST /v1/forwarders
```

```
PATCH /v1/forwarders
```

```
DELETE /v1/forwarders
```

```
{
  "name": "DHL Global Forwarding",
  "code": "DHLGF",
  "address": "Av. Example 123",
  "phone": "+54 11 4444-4444",
  "email": "ops@dhl.com",
  "companyCode": 4
}
```

## Respuesta esperada

```
{
  "success": true,
  "message": "Forwarder creado correctamente",
  "data": {
    "id": 1,
    "name": "DHL Global Forwarding",
    "code": "DHLGF",
    "address": "Av. Example 123",
    "phone": "+54 11 4444-4444",
    "email": "ops@dhl.com",
    "companyCode": 4
  }
}

```

---

## Reglas y validaciones

- Campos requeridos: `name`, `code`, `companyCode`


- `code` debe ser **único por **`companyCode`


- `email` validar formato si viene informado



---

## Errores

- **401**: token inválido o ausente


- **422**: validación de campos


- **409**: `code` duplicado para la empresa



---

## Criterios de aceptación

- Existe `POST /v1/forwarders`


- Persiste correctamente el forwarder


- No permite duplicar `code`


- Devuelve respuesta estándar `success / message / data`


- Mantiene paridad de estilo con `/v1/warehouses`



```
USE NewBytes_DBF;
GO

CREATE TABLE dbo.forwarders (
    id            INT IDENTITY(1,1) NOT NULL,
    name          VARCHAR(150)       NOT NULL,
    code          VARCHAR(20)        NOT NULL,
    address       VARCHAR(255)       NULL,
    phone         VARCHAR(50)        NULL,
    email         VARCHAR(150)       NULL,
    companyCode   INT                NOT NULL,

    createdAt     DATETIME2(0)       NOT NULL DEFAULT SYSDATETIME(),
    updatedAt     DATETIME2(0)       NULL,

    CONSTRAINT PK_forwarders PRIMARY KEY (id),

    CONSTRAINT UQ_forwarders_code_company
        UNIQUE (code, companyCode)
);
GO
```
