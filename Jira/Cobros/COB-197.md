## Descripción

Este recurso es basicamente una replica de [link](https://lioteam.atlassian.net/browse/COB-65)y puede incluso construirse sobre el mismo. Pero la diferencia es que ademas, lleva un payload o carga útil de cheques: Un arrays de id de cheques que se deben mover

Para esto:

Se debe generar una operación en `[MC_LOG_OPERACIONES]`

Mediante el recurso 

```
POST {{API_URL}}/v1/checksTrade
```

se envía la siguiente request 

```
[
  {
    description: 'Pases egresos', // puede no estar porque  esta transactionTypeId
    ammount: 2343.34,
    currency: 'Pesos',
    comment: 'Un comentario cualquiera',
    origin: 'Caja1',
    destiny: 'Dario',
    status: 'Abierto',
    satusId: 1,
    paymentMethodId: 15,
    transactionTypeId: 20,
    payloadChecks
    [
      idCheque,
      idCheque,
      idCheque,
      idCheque,
      idCheque,
      idCheque,
    ]
  }
  ]
```

Con esta informacion mas la informacion disponible en el back se debe confeccionar una fila en `[MC_LOG_OPERACIONES]` y en las tablas correspondientes de` [NEW_BYTES].[dbo].[MS_CHEQUES_MOVIMIENTOS]` y cambiar su estado correspondiente.
