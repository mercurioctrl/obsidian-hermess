## Descripción

Crearemos un endpoint a modo de “latido” que sirve para verificar de forma correcta la disponibilidad del servicio.

Esto permite conectarlo ademas a diferentes servicios como uptima kuma, uptime robot, etc

Seguiremos algo similar al proximo ejemplo generado por chatgpt pero adaptado a nuestras aplicaciones

```php
<?php

header('Content-Type: application/json');

// Configuración para la conexión a la base de datos
$dbHost = 'localhost';
$dbUsername = 'tu_usuario';
$dbPassword = 'tu_contraseña';
$dbName = 'tu_base_de_datos';

// Variables para el estado del sistema
$systemHealth = 'OK';
$httpStatusCode = 200; // Predeterminado a OK

// Intentar conectar a la base de datos
$mysqli = new mysqli($dbHost, $dbUsername, $dbPassword, $dbName);
if ($mysqli->connect_error) {
    $systemHealth = 'ERROR';
    $httpStatusCode = 503; // Servicio no disponible
} else {
    // Realiza chequeos adicionales aquí...
    // Por ejemplo, puedes verificar el uso de memoria, la carga del servidor, etc.
    // y actualizar $systemHealth y $httpStatusCode como sea necesario

    // Si todo está bien pero hay preocupaciones menores, establece un estado WARNING
    // $systemHealth = 'WARNING';
    // $httpStatusCode = 429; // Demasiadas solicitudes (como ejemplo de un warning)
}

// Establece el código de estado HTTP correspondiente
http_response_code($httpStatusCode);

$response = [
    'status' => $systemHealth,
    'timestamp' => date('c'),
    'databaseConnection' => $mysqli->connect_error ? 'Failed' : 'Successful',
    // Agrega más información del sistema según los chequeos realizados
];

// Asegúrate de cerrar la conexión a la base de datos si está abierta
if ($mysqli) {
    $mysqli->close();
}

echo json_encode($response);

```
