## Descripción

Se debe agregar una nueva sección llamada **“Percepciones”** dentro del menú **Impuestos**, donde ya se encuentran "Retenciones" y "Ganancias".


[attachment]
Esta sección debe mostrar los datos obtenidos desde el recurso:

```
GET {{API_URL}}/v1/perceptioniibb?type=arba&between=01-05-2025_31-05-2025&download=1
```

Según las historias vinculadas el recurso devuelve una tabla paginada con los campos de percepciones del período solicitado.

**Requerimientos de maquetado:**

- Mostrar en una tabla con los siguientes campos (columnas):

- Fecha de factura (`invoiceDate`)


- Factura (`documentLetter`-`branchNumber-documentNumber`)


- CUIT cliente (`clientCuit`)


- Nombre del cliente (`clientName`)


- Monto imponible (`taxableAmount`)


- Monto de percepción (`perceptionAmount`)


- Alícuota (%) (`rate`)


- Jurisdicción (`jurisdiction`)


- IVA (`ivaTax`)


- Otros conceptos (`otherConcepts`)





- Agregar paginación compatible con los datos del backend (`pagination.total`, `pagination.pageSize`, `pagination.current`).


- Agregar filtro por rango de fecha (utilizando el parámetro `between` en el formato `dd-mm-yyyy_dd-mm-yyyy`).


- Agregar selector de tipo de percepción (`type=arba/caba`, pero debe quedar preparado para futuros tipos).


- Mantener consistencia visual con las otras secciones de “Impuestos”.



## **Accionables de descarga**

Se deben agregar dos accionables para descargar esta misma informacion pero en los archivos exportables para las presentaciones juradas.

Los mismos toman el mismo endpoint con todos los filtros pero de la siguiente forma

```
GET {{API_URL}}/v1/perceptioniibb?type=CABA/ARBA&between=30-05-2025_31-05-2025&download=1
```

Es decir se manda `download=1` y no se mandan las paginaciones.
