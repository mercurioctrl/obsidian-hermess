## Descripción

```
PATCH {API_URL}/v1/adjustTo
```

```
{
  clienId: 3450,
  adjustToAmount: 1000,
  supportingText: "Este es un texto justificativo, minimo 50 caracteres",
}
```

Al ingresar el valor correcto del subtotal (`adjustToAmount`), el sistema analiza la situación actual de la cuenta corriente del cliente, incluyendo los movimientos registrados y los saldos pendientes. A continuación, se calculan las transacciones necesarias para igualar el subtotal deseado y se agrega una linea con la leyenda regularización, indicando quien fue el usuario que la realizo y una justificación de este movimiento administrativo.

Para esto trabajaremos sobre la tabla utilizada para las cuentas corrientes en [link](https://lioteam.atlassian.net/browse/COB-5) es decir `[NEW_BYTES].[dbo].[MC_CCORRIENTES_MOVIMIENTOS]` 

Tener en cuenta

Escribiremos la columna `[NEW_BYTES].[dbo].[MC_CCORRIENTES_MOVIMIENTOS].TR_CODIGO` con valores que provienen de `[NEW_BYTES].[dbo].[GL_TRANSACCIONES]` utilizando

**Créditos Varios **si el monto necesario para llegar a `adjustToAmount` es positivo, es decir un crédito.

Y **Débitos Varios **si el monto necesario para llegar a `adjustToAmount` es negativo, es decir un debito.

Finalmente utilizaremos el campo de observaciones para ingresar la justificación que debe tener un mínimo de 50 caracteres.
