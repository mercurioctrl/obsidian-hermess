## Descripción

Como usuario administrador, necesito un endpoint que cree un banco y automáticamente genere y asocie una cuenta bancaria, eliminando la necesidad de realizar dos pasos separados para poder operar con el banco.                                                                                       

- **Actualmente el flujo requiere:                                                                                                  **

- Crear el banco manualmente                                                                                                   


- Crear una cuenta bancaria pasando el bankId generado en el paso anterior 





  Con este nuevo endpoint ambas operaciones se ejecutan en una sola llamada.                                                      

- **Criterios de aceptación:**

- El endpoint recibe únicamente los datos del banco (descripción y opcionalmente companyCode)                                   


- Se crea el registro en la tabla BANCOS                                                                                        


- Se crea automáticamente el registro asociado en BA_BP_CAJA_CBANCARIAS usando el bankId generado                               


- La cuenta bancaria creada hereda la descripción del banco                                                                     


- Si companyCode es proporcionado, se valida que la empresa exista antes de crear                                               


- La respuesta retorna los datos del banco y la cuenta bancaria creada              


- El nombre del banco debe ser único                                            


- Si cualquier paso falla, ningún registro se persiste (transacción atómica) 







```
POST /v1/banks
```



`Payload:`

```
{                                                                                                                    {                                                                                                                               
  "description": "Galicia",                                                                                                   
  "companyCode": 5 -- opcional                                                                                                            
}
```

**companyCode es opcional**. Si no lo pasas, la cuenta se crea sin empresa asociada.                                      



**  Respuesta 201 — éxito: **                                                                                                        

```
{                                                                                                                               
      "success": true,                                                                                                            
      "message": "Banco y cuenta bancaria creados exitosamente",                                                                  
      "data": {                                                                                                                   
          "accountId": 15,                                                                                                        
          "description": "Galicia",                                                                                               
          "bankId": 12,                                                                                                           
          "bankName": "Galicia",                                                                                                  
          "companyCode": 5,                                                                                                       
          "companyName": "Empresa Ejemplo S.A."                                                                                   
      }                                                                                                                           
  }  
```

          

`Respuesta 422 — empresa no existe:`

```
{                                                                                                                               
    "success": false,                                                                                                           
    "comment": "La empresa indicada no existe",                                                                                 
    "status": 422                                                                                                               
}                                                                                                                               
```



`Respuesta 422 por duplicado:`

```
{
  "success": false,
  "comment": "Ya existe un banco con esa descripción",
  "status": 422
}
```



`Respuesta 422 — description faltante: `

```
{                                                                                                                               
    "success": false,                                                                                                           
    "comment": "El campo description es requerido",                                                                             
    "status": 422                                                                                                               
}  
```



`Respuesta 500 — fallo en la transacción (ej: error en DB):  `

```
{                                                                                                                               
    "success": false,                                                                                                           
    "comment": "Error al crear banco y cuenta bancaria",                                                                        
    "status": 500                                                                                                               
} 
```



 Si cualquier INSERT falla, la transacción hace rollback y ningún registro se persiste, ni el banco ni la cuenta.
