## Descripción

Actualmente, el campo `mp_pref_id` en la tabla `[LO].[dbo].[pedidosCabecera]` almacena valores determinados dependiendo del flujo utilizado en MercadoPago:

Parece ser que el `mp_pref_id` contiene un UUID que en realidad debería ser el `payment.id`.

Este refactor busca normalizar y clarificar el modelo de datos, separando correctamente los identificadores involucrados en los pagos con MercadoPago.

---

**Criterios de aceptación:**

- El campo `mp_pref_id` debe usarse **solo para guardar el ID de preferencia**.


- Se debe agregar un nuevo campo en la tabla de pagos: `mp_payment_id`, donde se guardará el `payment.id` devuelto por MercadoPago (el UUID).


- Se debe agregar otro campo nuevo: `mp_external_reference`, donde se guardará la `external_reference` seteada al momento de crear el pago.


- El servicio de pagos (`MercadoPagoPayment`) debe ser refactorizado para:

- Guardar correctamente el `mp_payment_id` después del `save()`


- Establecer y guardar el `external_reference` desde los datos de entrada (o generarla si no viene)




- Se debe agregar `mp_payment_id` a la tabla  para alojar en el lugar correcto lo que actualmente guardamos en `mp_pref_id`.



---

**Notas técnicas:**

- El cambio debe ser retrocompatible para no romper registros existentes ni flujos activos (ver imagen de como es la morfologia del ultimo registo y los anteriores).

Actualización:



- **Agregar el campo mp_payment_id a la tabla LO.dbo.pedidosCabecera**

- Tipo de dato: varchar(50). *igual a como se guarda hoy en la tabla LO.dbo.payment_gateway_transactions  el campos response_code*


- Este campo almacenará el ID de pago generado al procesar un pago mediante la API de Checkout de Mercado Pago. Cada vez que se genere un pago, el valor de mp_payment_id debe registrarse en este campo.




- **Crear Cron V4**

- Se utilizara para actualizar el estado de los pedidos en las tablas LO.dbo.pedidosCabecera y LO.dbo.payment_gateway_transactions.


- Condiciones:

- Solo aplicará a los pedidos existentes en LO.dbo.payment_gateway_transactions que correspondan a pagos procesados mediante la API de Checkout de Mercado Pago.


- El cron únicamente actualizará el campo mp_payment_status en LO.dbo.pedidosCabecera, el cual se utiliza en la sección "Mis Compras"


- Actualizara los estados en la tabla LO.dbo.payment_gateway_transactions. 









- Ejemplo de consulta a la API  a  MP:



```
curl --location --request GET 'https://api.mercadopago.com/v1/payments/{mp_payment_id}' \
--header 'Authorization: Bearer jwt...'
```
