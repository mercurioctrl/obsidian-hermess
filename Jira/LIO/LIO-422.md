## Descripción

Desarrollar el endpoint en v4


```
POST {API_URL}/v4/paymentIntents/create
```


Registra una nueva intención de pago en el sistema, almacenando su estado inicial como `pending` o similar. Este endpoint será el primer paso del flujo de intenciones de pago, reservando un ID único para completarlo posteriormente.

**Objetivo**:
Proveer un recurso que reciba la información mínima de una intención de pago, la almacene en la base de datos y devuelva un identificador único (UUID) junto al estado inicial.

---

**Requerimientos funcionales**:

- **Entrada mínima aceptada**

- `amount` *(decimal, opcional)* — Monto inicial de la intención (puede ser NULL si no se conoce todavía).


- `currency` *(string, opcional)* — Moneda (default `ARS`).


- `typeOperation` *(string, requerido)* — Ej: `pagar_orden`, `cargar_wallet`, `cobro_externo`.


- `origin` *(string, requerido)* — Ej: `libreopcion`, `sistema_interno`.


- `referenceNumber` *(string, opcional)* — Identificador externo (orden, ticket, etc.).




- **Proceso**

- Insertar registro en `PaymentIntents` con:

- `status` = `pending`


- `createdAt` con la fecha/hora actual.


- `updatedAt` NULL inicialmente.




- Registrar en `PaymentLogs` un evento con mensaje `"Payment intent created"` y estado `pending`.




- **Salida esperada** (HTTP 201 Created):

```
{   
"id": "uuid",   
"status": "pending",   
"createdAt": "2025-08-12T10:00:00Z" 
} 
```




- **Validaciones**

- `typeOperation` y `origin` obligatorios.


- `currency` debe ser un código válido ISO 4217 (ej. `ARS`, `USD`).


- `amount` si viene, debe ser mayor a 0.




- **Respuestas de error**

- 400 Bad Request: Parámetros faltantes o inválidos.


- 500 Internal Server Error: Fallo interno en la creación.





---

**Tablas involucradas**:

- `PaymentIntents` (insert principal).


- `PaymentLogs` (insert log de creación).
