## Descripción

Registra el navegador actual del usuario para recibir notificaciones push vía FCM. Debe llamarse luego de que el usuario otorga permisos de notificación y se obtiene el token FCM. Es idempotente: si el navegador ya estaba registrado, actualiza el token y metadata. Permite que un mismo usuario tenga múltiples navegadores registrados simultáneamente.

El `deviceId` debe generarse en el cliente la primera vez y persistirse en `localStorage`. Si no existe, se crea con `UUID` antes de llamar al endpoint.



`POST /v4/push/devices`

Requiere JWT válido en el header `Authorization: Bearer <token>`.

```json

    "token": "fGx1yZ3aB:APA91bHk...",                                                                                             
    "environment": "production",                        
    "appVersion": "3.14.0",
    "buildNumber": null,
    "osVersion": null,
```
