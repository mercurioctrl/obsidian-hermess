## Descripción

Implementar protección por **token estático** para el endpoint de sincronización, evitando el uso del middleware de autenticación tradicional (user/password).

El recurso debe ejecutarse **únicamente** cuando se reciba un parámetro `token` por querystring que **coincida exactamente** con el valor configurado en el `.env`.

- Si el token recibido es válido → se ejecuta el proceso de sync.


- Si el token es inválido o no se envía → responder `401 Unauthorized` y no ejecutar ninguna acción.



**Endpoint**

```
POST {API_URL}/v4/yt/shorts/syncUp?label={label}&token={token}
```

**Regla de validación**

```
token (request) === SYNCUP_TOKEN (.env)
```
