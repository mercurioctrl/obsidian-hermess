## Descripción

listar compras y poder filtrar por
`orderStatus` : “cancelado” | “preparando” | ….

`paymentStatus`: “`pago_aprobado`"
`shippingStatus`: “envio pendiente” | “en camino” | “Entregado”



Se debe tambien crear un recurso para listar los estados

```
        orderStatus:
            "Cancelado"
            "Recibimos tu pedido"
            "Listo para retirar"
            "Listo para enviar"
            "Entregado"
            "Enviado"
            "Sin Estado"
            "Pedido cancelado"
            "Entregado, con cambios"
            "Enviado, con cambios"


          paymentStatus: 
            "Pago aprobado"
            "Verificando comprobante de pago
            "Esperando Pago"
            "Pendiente de pago"
            "Autorizando Pago"
            "Pago acreditado"
            "pago aprobado, devuelto parcialmente"

          shippingStatus:
            "Envio Pendiente" 
            "En camino" 
            "Entregado"
            "Preparando envio"
```

---

### 📌 Detalles técnicos

- La lógica actual para traducir un `orderStatusId`, `medioEnvioId` y `lanula` en un estado legible está en `App\Dto\OrderStatus\OrderStatusDto`.


- Se deben testear diferentes combinaciones de esos valores para cubrir todos los posibles casos.


- El endpoint podría ser algo como:



```
GET {APIv4_URL}/api/orderStatus
```

Retorna:

```
[
  "Recibimos tu pedido",
  "Listo para retirar",
  "Listo para enviar",
  "Enviado",
  "Entregado",
  "Pedido Cancelado",
  "Enviado, con cambios",
  "Entregado, con cambios",
  "Cancelado",
  "Sin Estado"
]

```

### 🎯 Criterios de Aceptación

- Se debe crear una clase repositorio (`OrderStatusRepository`) que retorne un listado único con todos los posibles estados que devuelve el DTO `OrderStatusDto`.


- Los estados deben ser los mismos que devuelve el método `OrderStatusDto::determineOrderStatus()`.


- No debe haber duplicados.


- El repositorio debe ser reutilizable.


- Debe exponerse un endpoint en la API que devuelva este listado en formato JSON.



**Código ostentativo para lograr obtener los estados combinados según el metodo original**

```
<?php

declare(strict_types=1);

namespace App\Repository\OrderStatus;

use App\Dto\OrderStatus\OrderStatusDto;

final class OrderStatusRepository
{
    public static function getAllStatuses(): array
    {
        // Set de combinaciones que queremos testear para mapear todos los estados
        $combinations = [
            [1, null, null],
            [2, null, null],
            [2, 3999, null],
            [2, 1, null],
            [3, 1, null],
            [4, 1, null],
            [9, null, null],
            [10, null, null],
            [11, null, null],
            [12, null, null],
            [13, null, null],
            [13, 3999, null],
            [14, 1, null],
            [14, 3999, null],
            [15, null, null],
            [16, 1, null],
            [16, 3999, null],
            [0, null, null],
            [1, null, 1], // lanula = 1 (cancelado)
        ];

        $statuses = [];

        foreach ($combinations as [$orderStatusId, $medioEnvioId, $lanula]) {
            $status = OrderStatusDto::determineOrderStatus($orderStatusId, $medioEnvioId, $lanula);
            $statuses[$status] = true;
        }

        return array_keys($statuses);
    }
}
```
