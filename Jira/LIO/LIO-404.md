## Descripción

La idea es integrar en el mismo recurso de asistencia de compatibilidad diferentes modelos, lo haremos en este caso para el modelo de deepseek.

El funcionamiento es exactamente el mismo, pero el research consiste en hacerlo multi-modelo y utilizar un parámetro dentro de las variables  de entorno para poder cambiar entre uno u otro mediante el parámetro `COMPATIBILITY_MODEL`

Se deja mas abajo un token para el nuevo modelo

`DEEPSEEK_API_KEY=sk-607dc8ff032a4573a5a597be5cac21ef`

`COMPATIBILITY_MODEL={DEEPSEEK/GPT4}`
