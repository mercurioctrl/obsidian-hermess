## Descripción

Se debe verificar que ambos recursos 

```
POST /v4/syncUp/items
```

```
PATCH /v4/syncUp/items
```

respeten el flag `ocultar_lo = 1` proveniente de `[NewBytes_DBF].[dbo].[articulo]`,
Para que los ítems marcados como ocultos no se importen ni se actualicen en el catálogo, y así evitar que queden visibles aunque no tengan stock o estén discontinuados.

**Contexto:**
Actualmente se detectó que ítems con `ocultar_lo = 1` siguen apareciendo en el catálogo. Si bien visualmente no tienen stock, siguen listados.


**Criterios de aceptación:**

- El endpoint **POST /v4/syncUp/items** debe ignorar ítems cuyo `ocultar_lo = 1`.


- El endpoint **PATCH /v4/syncUp/items** debe evitar actualizar ítems que actualmente tengan `ocultar_lo = 1` (o bien marcar el ítem como oculto dentro del sistema si ya fue importado).


- En caso de que técnicamente sea complejo aplicar la lógica en PATCH (por ejemplo, por estructuras internas), se evaluará una alternativa razonable como marcar el ítem como inactivo o pasarlo a estado "oculto".



**Notas técnicas:**

- La columna `ocultar_lo` se encuentra en la tabla `[NewBytes_DBF].[dbo].[articulo]` y representa un flag binario (1 = ocultar, 0 = mostrar).
