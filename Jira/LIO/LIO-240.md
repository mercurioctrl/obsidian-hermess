## Descripción

### 

Devuelve el saldo actual de la billetera del usuario autenticado en **ARS (pesos argentinos)**. Se puede filtrar entre saldo disponible (`available`) y saldo pendiente (`pending`).

```
GET {API_V4}/wallet/balance?type={available|pending}
```

### **Parámetros de consulta (Query Params)**

Parámetro

Tipo

Requerido

Descripción

`type`

`string`

No

Filtra el saldo: `"available"` para saldo disponible, `"pending"` para saldo pendiente.

### **Ejemplo de Respuesta Exitosa (200 OK)**

**Sin filtro (ambos saldos)**

```
{   
"userId": "user_789",   
"available": 150000.00,   
"pending": 25000.00 
} 
```

**Solo saldo disponible**

```
{   
"userId": "user_789",   
"available": 150000.00 
} 
```

**Solo saldo pendiente**

```
{   
"userId": "user_789",   
"pending": 25000.00 
}
```

Para saber que movimientos están marcados como pendientes agregaremos una columna en `[NEW_BYTES].[dbo].[MC_CCORRIENTES_MOVIMIENTOS]` llamada `pending` que puede ser NULL y no es obligatoria. Solo cuando es TRUE, es importante. Todos los demas valores quiere decir que forma parte de `available`
