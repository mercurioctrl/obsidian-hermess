## Descripción

Conectar el flujo de modificación de precios con el endpoint de **Inventories v4**, reemplazando la lógica previa y centralizando la actualización en el nuevo recurso.

---

## Endpoint

```
PATCH /v4/inventories/products/{productId}/list
```

### Ejemplo de request

```
curl --location --request PATCH 'http://localhost:8097/v4/inventories/products/641009/list' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer {token}' \
--data '{
    "price": 4983297
}'
```

---

## Regla de negocio principal

El **precio nunca puede ser menor al costo del producto**.

Esta validación debe existir en **dos niveles**:

- Validación de UI (input)



---

## Comportamiento esperado

### Validación en Frontend

- El input de precio no debe permitir ingresar valores menores al costo.


- Si el usuario intenta ingresar un valor menor:

- Se debe bloquear la acción.


- Se debe mostrar un mensaje claro y explícito.





Ejemplo de mensaje:

```
El precio no puede ser menor al costo del producto.
```

---

### Validación en Backend (obligatoria)

Aunque el frontend valide, el backend debe volver a validar siempre:

Si `price < cost`:

- La operación debe rechazarse.


- Debe devolverse error de validación.
