## Descripción

[attachment]


El objetivo es permitir que cada landing page o sitio de un cliente pueda registrar sus suscriptores de newsletter dentro del sistema centralizado.
Cada cliente utilizará su propia **apiKey**, almacenada en el archivo `.env`, evitando valores hardcodeados y facilitando la rotación del token.

### **Implementación**

El frontend deberá enviar las suscripciones al recurso:

```
POST {API_URL}/partner/newsletter/subscriptions
```

#### **Headers obligatorios**

```
X-Client-ApiKey: {apiKeyDelCliente}
Content-Type: application/json

```

#### **Payload esperado**

```
{
  "siteOrigin": "https://landing-cliente.com",
  "email": "suscriptor@example.com",
  "name": "María López",
  "subscriptionSource": "popup-home"
}

```

### **Descripción funcional**

- `siteOrigin`: URL del sitio o landing desde la cual se realiza la suscripción.


- `email`: correo electrónico del suscriptor.


- `name`: nombre del suscriptor (opcional según el cliente).


- `subscriptionSource`: identificador del origen de la suscripción (por ejemplo: `popup-home`, `footer-form`, `landing-campania`).
