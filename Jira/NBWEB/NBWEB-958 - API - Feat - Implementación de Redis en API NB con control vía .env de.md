## Descripcion

Vamos a implementar Redis como sistema de cacheo para la API de NB, con el objetivo de mejorar el rendimiento y disminuir la carga en consultas frecuentes.

---

### Alcance:

- Integrar Redis en la API de NB.


- Agregar variables de entorno para:

- Activar o desactivar el uso de cache: `USE_REDIS_CACHE=true|false`


- Configurar el tiempo de cacheo (TTL) en segundos: `REDIS_CACHE_TTL=1800` (valor inicial: 30 minutos)




- Cuando el cache esté deshabilitado (`USE_REDIS_CACHE=false`), la API debe responder en tiempo real sin utilizar Redis.


- El TTL debe aplicarse correctamente a todos los endpoints definidos, y debe poder modificarse sin cambios de código (solo actualizando el `.env` y reiniciando la app).


- El cache debe ser sensible a los parámetros de cada solicitud para evitar respuestas incorrectas.



---

### Endpoints a cachear inicialmente:

- `GET /v1/`


- `GET /v1/{busqueda}`


- `GET /v1/banners/{idSeccionBanner}`


- `GET /v1/?ultimos_ingresos={parametro}`


- `GET /v1/item/{idItem}`


- `GET /v1/autoGeneratedDescription/{idItem}`


- `GET /v1/brands`


- `GET /v1/categories`



---

### Variables de entorno nuevas:

`USE_REDIS_CACHE=true `
`REDIS_CACHE_TTL=1800 `

---

### Criterios de aceptación:

-  Redis queda correctamente integrado en el proyecto.


-  Existe una variable de entorno `USE_REDIS_CACHE` que habilita o deshabilita el uso de cache.


-  Existe una variable de entorno `REDIS_CACHE_TTL` que define el tiempo de expiración del cache en segundos.


-  Todos los endpoints listados cachean su respuesta durante el tiempo especificado por `REDIS_CACHE_TTL` cuando el cache está habilitado.


-  Cuando `USE_REDIS_CACHE=false`, todos los endpoints responden en tiempo real sin utilizar Redis.


-  Las respuestas cacheadas consideran los parámetros dinámicos para evitar colisiones de key.


-  Se realiza testing manual cambiando los valores del `.env` para verificar que el sistema responde correctamente tras reiniciar la aplicación.


-  Redis no debe interferir con el comportamiento habitual de la API cuando está desactivado.
